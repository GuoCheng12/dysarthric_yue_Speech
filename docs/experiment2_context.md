# Experiment 2 Context

## Current Direction

Experiment 1 is complete: Qwen3-ASR zero-shot inference has been run on `vlink_data_raw`.

Experiment 2 is now defined as cleaned real-data-only pooled fine-tuning:

- First clean the data automatically.
- Exclude Q&A prompt-answer mismatch rows from the first training run.
- Keep short-word / isolated-phrase rows for analysis only, not the main pooled read-sentence training.
- Run one pooled fine-tune before complex patient speech profiling.
- Evaluate by strata, not only overall CER.

Patient speech profiling, phenotype clustering, augmentation planning, and patient-specific adapters are deferred until after the first pooled fine-tuning result.

## Required Evaluation Strata

Report zero-shot vs fine-tuned CER and delta CER for:

- overall
- easy: `zero_shot_cer < 0.2`
- medium: `0.2 <= zero_shot_cer < 0.5`
- hard: `zero_shot_cer >= 0.5` or `zero_shot_critical=true`
- per speaker
- per disease_tag
- read_sentence vs short_word
- duration buckets

Interpretation:

- Overall and hard improve: continue with data-size curve, augmentation ablation, speaker-balanced training.
- Overall improves but hard does not: move to patient speech profiling, hard-speaker oversampling, patient-specific LoRA/adapter, phone/tone targeted augmentation.
- Overall improves but some speakers regress: test speaker-balanced sampler, disease/phenotype-aware training, patient-specific adapters.
- Nothing improves: debug cleaning quality, prompt leakage, learning rate, epoch count, overfitting, and reference normalization before augmentation.
